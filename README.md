# Тренажёр · Испытания технологических машин и оборудования

Веб-тренажёр для подготовки к экзамену по дисциплине «Испытания технологических машин и оборудования». 211 вопросов из перечня 2026 года с вариантами ответов, объяснениями и ссылками на учебник Попова, Полищука, Решетова.

## Как запустить

### Вариант 1 — самый простой
1. Скачайте всю папку `trainer/` себе на компьютер.
2. Откройте файл `index.html` двойным щелчком.
3. Тренажёр откроется в браузере (Chrome, Firefox, Edge, Safari — без разницы).

**Не нужно ничего устанавливать, никакого сервера, никакого интернета.** Всё работает локально в браузере.

### Вариант 2 — через локальный сервер (опционально)
Если двойной клик не сработал (некоторые браузеры режут локальные `file://`-страницы):
```bash
cd trainer/
python3 -m http.server 8000
```
И открыть в браузере: `http://localhost:8000`

## Что умеет

### Режимы тренировки

1. **Все вопросы** — все 211 вопросов в случайном порядке. Для полного прогона.
2. **Экзаменационная сессия** — 30 случайных вопросов, точная имитация экзамена. Оценивается по реальной шкале:
   - **29–30 правильных → «отлично» (5)**
   - **28 правильных → «хорошо» (4)**
   - **27 правильных → «удовлетворительно» (3)**
   - **< 27 → неудовлетворительно**
3. **Работа над ошибками** — собираются вопросы, на которые отвечали неверно последний раз. Если таких нет, добавляются вопросы с низкой точностью.
4. **По разделам** — 9 разделов (Ударные стенды — основы / Ударные стенды — динамика / Виброиспытания / Центрифуги / ВКУ центрифуг / Прочие стенды / Аттестация и метрология / Электроприводы и IP / Лебёдки и датчики).

### Логика теста

- На каждый вопрос — 4 варианта ответа (А, Б, В, Г).
- **Варианты перемешиваются каждый раз** — нельзя запомнить «правильный ответ всегда Б».
- После клика на вариант — мгновенная подсветка: зелёный = правильный, красный = ошибка.
- Сразу под вопросом появляется **объяснение** с цитатой и ссылкой на раздел учебника.

### Ссылки на источник (новое в версии 3.0)

- Если ответ есть в учебнике, **цитата кликабельна**: по нажатию открывается встроенный просмотрщик с нужной страницей учебника и **подсвеченным фрагментом** (через PDF.js). Под объяснением — кнопка «📖 Открыть в учебнике · стр. N».
- Если ответа в учебнике нет, но есть сноска на внешний источник (ГОСТ, статья, Википедия) — кнопка «🔗 Открыть источник» открывает страницу во вкладке **с автоподсветкой фразы** (Text Fragments). Для документов за пейволлом (например, ГОСТы на cntd.ru) откроется просто страница без подсветки.

### Статистика

- Сохраняется в `localStorage` браузера (между сессиями).
- В шапке всегда видны: изучено / верных / ошибок / точность.
- На главной странице — дашборд по разделам:
  - точность по каждому разделу (с цветной полосой: красная если <50%, жёлтая 50–75%, зелёная >75%);
  - список вопросов, требующих повторения (где последний ответ был неверным).
- Кнопка «Сбросить статистику» — внизу главной.

## Структура файлов

```
trainer/
├── index.html              ← главный файл, открывать его
├── lectures.pdf            ← учебник (нужен для кнопки «Открыть в учебнике»)
├── questions_part1.js      ← вопросы 1–50 (Ударные стенды основы)
├── questions_part2.js      ← вопросы 51–110 (Динамика + Виброиспытания)
├── questions_part3.js      ← вопросы 111–170 (Центрифуги, ВКУ, прочие)
├── questions_part4.js      ← вопросы 171–211 (Аттестация, IP, датчики)
└── README.md               ← этот файл
```

> ⚠️ Файл `lectures.pdf` обязателен для функции открытия учебника по клику на цитату. Если открывать тренажёр как `file://` двойным щелчком, некоторые браузеры блокируют загрузку PDF — тогда запустите через локальный сервер (Вариант 2 выше).

## Важные нюансы по содержанию

Часть вопросов перечня **не имеет точного ответа в учебнике** (особенно про КСП5, ВКУ, ПИД-регулятор, датчики, гладкий статор, стенды качки). Для них ответы даны по общетехническим знаниям, и это явно помечено в объяснении словом **«ВНИМАНИЕ»** или **«в учебнике прямо не описано»**.

На таких вопросах — обязательно сверься с лектором или конспектом, чтобы не учить версию, которая может расходиться с эталоном преподавателя.

## Технические детали

- 100% статический HTML+CSS+JS, без зависимостей.
- Никаких внешних запросов, кроме шрифтов Google Fonts (можно работать и без них).
- Без cookies, без аналитики, без трекинга.
- Адаптивная вёрстка — работает на мобильных.
- localStorage сохраняет статистику между сессиями (в пределах одного браузера).

Удачи на экзамене!
